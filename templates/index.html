Sobre o projeto

O que é o Olist Tiny Code Auth?

Uma ponte minimalista entre você e o fluxo OpenID Connect do Olist Tiny. Construída para ser direta, transparente e sem fricção.

100% serverless

Roda em funções serverless no Vercel. Sem banco de dados, sem cache, sem persistência de qualquer tipo.

Sem armazenamento

O client_id informado e o authorization code retornado não são gravados em lugar nenhum.

Open source

Código aberto sob licença MIT. Audite, contribua ou faça fork. Desenvolvido e mantido pela CTEC.

State anti-CSRF

Cada requisição gera um token state único via CSPRNG, protegendo o fluxo contra ataques CSRF.

Passo a passo

Como funciona

Quatro passos do formulário até o authorization code na sua tela.

Informe o client_id

Cole o client_id do seu app registrado no Olist Tiny no formulário abaixo.

Redirecionamento OIDC

A aplicação monta a URL do Tiny com scope=openid e state único, e redireciona você.

Login no Tiny

Autentique-se com suas credenciais do Olist Tiny e autorize o acesso ao aplicativo.

Copie o código

O Tiny retorna o authorization code, que é exibido pronto para copiar.

Configuração

Como configurar no Olist Tiny v3

Antes de usar, registre a URL de Redirecionamento no cadastro do seu aplicativo no painel do desenvolvedor Olist.

  1. Acesse o painel do desenvolvedor do Olist Tiny e abra o cadastro do seu aplicativo (ou crie um novo).
  2. No campo URL de Redirecionamento (Redirect URI), informe exatamente:
    https://olist-tiny-teste2.vercel.app/callback
    Em desenvolvimento local, use http://localhost:5000/callback.
  3. Salve o cadastro. O client_id e o client_secret serão gerados pelo painel.
  4. Volte para cá e gere o authorization code. Em seguida, troque-o pelo access_token no endpoint de token do Tiny, conforme a documentação oficial.
